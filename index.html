Foundation

Claude.ai — cánh cửa #1

Truy cập tại claude.ai. Chat đa năng cho công việc văn phòng hàng ngày.

  • Projects — folder chứa tài liệu nền (brief khách, brand guideline). Claude nhớ context xuyên session.
  • Artifacts — Claude tạo HTML, React, SVG, Markdown render trực tiếp xem được.
  • File upload — kéo thả PDF/Excel/ảnh, Claude đọc và phân tích.
  • Connectors (MCP) — kết nối Gmail, Drive, GitHub, Notion để Claude lấy data thật.

Khi nào dùng Claude.ai?

  • Việc one-off: viết, dịch, tóm tắt, brainstorm
  • Phân tích 1 file PDF/Excel cụ thể
  • Tạo prototype UI nhanh (Artifact)
  • Hỏi đáp với tài liệu nội bộ (Projects)

KHÔNG dùng cho:

  • Sửa cả codebase đa file (dùng Claude Code)
  • Workflow tự động lặp lại (dùng Cowork)

Foundation

Cowork mode — đồng nghiệp AI

App desktop (macOS/Windows) tự động hóa file & task cho người không code. "Delegate, don't dialogue".

Khác chat truyền thống ở 4 điểm

  • File native — đọc/ghi .docx, .xlsx, .pptx thật
  • Reasoning — tự lên plan nhiều bước, không cần bạn nhắc
  • Browser tool — tương tác web khi cần
  • Connector — Gmail, Slack, Drive, JIRA

Bài test "có hợp với Cowork không?"

  • Cần đụng vào files/folders thật?
  • Cần dùng Gmail/Drive/Slack?
  • Tạo file output hoàn chỉnh (.pptx, .xlsx)?
  • Nhiều bước hoặc lặp lại thường xuyên?

≥ 2 câu CÓ → dùng Cowork. Còn lại → dùng Chat.

Prompting

Giải phẫu một prompt tốt

Prompt = giao việc cho đồng nghiệp giỏi nhưng mới. Càng rõ — càng đúng.

[ROLE] Bạn là chuyên viên content B2B [CONTEXT] Kaopiz là công ty IT outsourcing, khách mục tiêu CTO/Tech Lead Nhật. [TASK] Viết 1 post LinkedIn về AI offshoring. [CONSTRAINTS] - 200-250 từ tiếng Việt - Hook thuộc nhóm "contrarian" - Có 1 metric cụ thể - CTA: comment để nhận case study [FORMAT] Trả về 3 phương án, mỗi cái có hook + body + CTA tách rõ.

5 thành phần cốt lõi

①
RoleClaude đóng vai gì? Bằng cấp/kinh nghiệm gì?
②
ContextBối cảnh, audience, mục tiêu kinh doanh
③
TaskMột câu lệnh hành động rõ ràng
④
ConstraintsĐộ dài, tone, ngôn ngữ, điều cấm
⑤
FormatBullet, table, JSON, số phương án...
Prompting

6 nguyên tắc "prompt chất lượng cao"

🎯
1. Cụ thể, đo được

"Viết bài 250 từ, hook contrarian, kết bằng câu hỏi" thay vì "viết bài hay hay".

📚
2. Đưa ví dụ

2-3 ví dụ output mong muốn (few-shot) giúp Claude bắt tone & style ngay.

🧩
3. XML tag chia khối

Dùng <context>, <task>, <example> — Claude phân biệt rõ phần nào.

🪜
4. Nghĩ từng bước

"Hãy nghĩ từng bước trước khi trả lời" — task suy luận tăng độ chính xác.

🔁
5. Lặp & tinh chỉnh

"Hãy ngắn hơn 30%", "Bỏ phần X" — đừng làm lại từ đầu.

📤
6. Chỉ định format

JSON? Markdown? 3 phương án? Nói rõ — không thì Claude tự đoán.

Prompting

Trước & Sau — sức mạnh của prompt rõ ràng

❌ Prompt yếu
Viết email cảm ơn khách

Vấn đề: Claude không biết khách là ai, ngôn ngữ, độ dài, tone, ký tên ai. Output chung chung, mỗi lần khác nhau, phải sửa nhiều.

✅ Prompt tốt
Viết email tiếng Nhật cảm ơn khách hàng (CTO công ty Z) đã ký HĐ offshoring 6 tháng với Kaopiz. Tone: trang trọng nhưng ấm áp. Độ dài: ~150 từ. Cấu trúc: cảm ơn → cam kết chất lượng → bước tiếp theo (kickoff). Ký tên: Sales Lead Nguyễn A. Trả về: subject + body.

Kết quả: output nhất quán, đúng tone, dùng được ngay — chỉnh tối đa 5%.

Prompting

8 lỗi phổ biến — và cách tránh

  • Prompt 1 dòng — quá ít context, output chung chung
  • Hỏi xong copy luôn — không đọc, không tinh chỉnh
  • Một chat đa task — Claude lẫn lộn context cũ
  • Không feedback — sai mà không bảo "sai chỗ X"
  • Tin tuyệt đối — số liệu, tên người vẫn cần verify
  • Không upload tài liệu nguồn — bắt Claude đoán thông tin nội bộ
  • Sợ prompt dài — prompt 500 từ ra output tốt hơn prompt 5 từ
  • Không lưu lại — mỗi lần làm lại từ đầu, không build template

Mẹo Pro: sau mỗi prompt thành công, dán vào file my-prompts.md. 3 tháng sau bạn sẽ có "kho prompt" cá nhân — đó là tài sản thật. Khi đủ thuần, hãy đóng gói thành Skill (xem phần Cấp độ 3).

BÀI 05

Review code đơn giản — bug, style, perf

DỄ ⏱ 8 phút 🛠 Claude.ai 🎯 Dev junior / mid

Tình huống

Trước khi mở PR, paste function lên Claude — bắt được lỗi cơ bản & gợi ý refactor mà reviewer hay góp ý. Tránh được 60% review comments.

📥 Input mẫu
function login(email, password) {
  const user = db.users.find(u => u.email === email);
  if (user && user.password === password) {
    return { token: user.id, role: user.role };
  }
  return null;
}

Quy tắc

  • Paste tối đa 1 file/function — context tập trung
  • Nói rõ tech stack & convention
  • Yêu cầu output có severity (Critical/Warning/Nit)
  • Bảo Claude show diff đã sửa, không chỉ giải thích

Prompt mẫu

Bạn là senior reviewer team React. Convention: TypeScript strict, ESLint Airbnb, React 18, hooks-only. Review function dưới và trả về: ## Critical (sai logic / bug bảo mật) ## Warning (best practice) ## Nit (style, naming) Mỗi mục: vấn đề + đoạn code minh họa + bản fix. Cuối: viết lại function với mọi fix đã áp dụng (TypeScript). [paste code function]

Công thức Description vàng

Description quyết định ai dùng Skill và khi nào dùng. Sai 1 dòng — Skill ngủ đông mãi mãi.

✅ GOOD

"Tạo báo cáo tuần dạng BLUF [What] khi user upload file Excel KPI hoặc yêu cầu tóm tắt tuần [When]."

Lý do: rõ ràng, cụ thể, kích hoạt chính xác đúng lúc. Có cả What và When.

❌ BAD

"Giúp làm báo cáo cho các dự án."

Lý do: quá chung chung, dễ over-trigger trong các ngữ cảnh không liên quan, hoặc không trigger.

[Action] + [Object] + [Trigger condition] Ví dụ: "Tạo proposal Nhật" + "khi khách hỏi báo giá team offshore" "Review PR" + "khi user paste link GitHub PR" "Generate weekly report" + "khi user upload timesheet hoặc nói summarize tuần" Mẹo: - Liệt kê 5 biến thể câu lệnh user thật trong description - Thêm "proactively" để Claude tự kích hoạt khi thấy workflow phù hợp - Đo: trigger rate > 90%, 0 false positive

Best practices

10 thói quen của Claude power user

  • Mỗi task = 1 chat mới — context không rò rỉ
  • Lưu prompt thắng vào file, biến thành template, sau đó thành Skill
  • Upload tài liệu nguồn thay vì paste vào prompt
  • Hỏi Claude tự critic — "phương án này có lỗ hổng gì?"
  • So sánh nhiều phiên bản — yêu cầu 3 cách viết để chọn
  • Verify số liệu & tên — Claude vẫn có thể hallucinate
  • Cá nhân hóa Project với brand voice + ví dụ
  • Dùng tool đúng level — đừng dùng Claude.ai cho task lặp 100 lần

  • Chia sẻ & học từ team — best prompt nội bộ tuần
Cảnh báo

Đừng làm những điều này — anti-patterns

🚫
Vibe coding mù

Claude tạo code → push thẳng prod, không đọc, không test. Bug âm thầm rất tốn kém về sau.

🔓
Paste data nhạy cảm

Paste API key, password, PII khách hàng vào chat free tier. Dùng Team/Enterprise + redact trước.

📜
Tin số liệu Claude bịa

"Theo nghiên cứu Stanford 2024…" — có thể hallucination. Bắt Claude trích link, verify bằng web search.

🎭
Không ghi nguồn AI

Bài viết AI viết → đăng tên cá nhân. Vi phạm policy nội bộ Kaopiz và đạo đức nghề nghiệp.

🔄
Auto-loop không giới hạn

Agent gọi API hàng ngàn lần/ngày. Set max_steps, budget, alert — tránh "AI bill" đột biến.

🪞
Phụ thuộc tuyệt đối

Mọi quyết định đều hỏi AI, không build skill cá nhân. AI là đòn bẩy — không thay thế tư duy.
